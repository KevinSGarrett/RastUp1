K\) Scheduling, Chunking & Idempotency

Chunk size defaults (e.g., 500 users; 2,000 bookings; 10,000 index
docs).

Pacing: interbatch delay (e.g., 2--10s), offpeak windows (cityaware).

Idempotency: each batch writes a job_id + batch_id footprint; retries
skip alreadydone rows.

Resume: pick up from last successful batch; Stop is safe; Requeue failed
only.

Rate limits: external API calls (social, PSP) throttled; exponential
backoff.
