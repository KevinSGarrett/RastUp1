C\) Inbox: layout & organization

Layout:

Left pane: Folders + thread list

Center: Messages

Right pane: Project Panel (context & actions)

Folders:

All, Unread, Starred, Inquiries, Invites, Bookings, Disputes, Archived,
Spam

Filters: by Role (Model, Photographer, ...), by Status (Awaiting reply,
Payment pending), by City/Date, by Has files

Bundles: Threads are bundled by counterparty; expand to see separate
rolescoped conversations

Thread list item:

Avatar + Name, Role chip, Type chip (Inquiry / Booking / Invite), Status
chip, last message preview, unread count, date, read receipt icon if
your last message was read

Thread search:

Keyword search, file type filters, date range, role filter

Message requests:

Firsttime contacts land in Requests (under Inquiries) with Accept /
Decline / Block.

Bypass to Inbox automatically if sender meets your contact filters
(e.g., IDverified, inradius, budget disclosed).

D\) Compose & credits (spamsafe by design)

New conversation credits (monthly) per account, with bonus credits from
verification badges (as defined in §1.1).

Unlimited replies in existing threads and unlimited messaging for
confirmed bookings.

Contact filters: Talent can require IDverified only, budget disclosure,
or date provided to message.

Quick replies & snippets: Providers can save templated responses; we
autosuggest based on context ("Thanks, I'm available that day---package
options attached").

Message booster: For buyers, optional oneoff priority ping
(limited/charged) to nudge highdemand talent.

E\) Thread header & Project Panel (right pane)

Thread header (always visible):

Counterparty name + avatar; Role chip; Type chip; Status

Booking summary bar when applicable: date/time • location • package •
price • escrow status • policy

Project Panel tabs (Booking threads):

Summary: package, extras, license, price, escrow status, timeline
(proposal → accepted → shoot → delivery → review)

Schedule: date/time, buffers, reschedule button, ICS export

Location: map pin, parking notes; travel terms; share location (toggle)

Participants: buyer, provider, assistants (optional), MUA/stylist tags
(FYI only); roles & contact via chat

Brief: project description + structured brief fields (theme, wardrobe,
references)

Moodboard: pinned images/links, dragreorder, annotate with comments

Shot List: checklist of shots; statuses (planned, in progress, done)

Call Sheet: autogenerated PDF (time, location, contacts, setup notes,
schedule blocks, wardrobe list)

Files & Deliverables: proofs, finals, RAW (if allowed); preview &
approvals; watermarked previews until marked "accepted"

Contracts: booking agreement, model release, usage license, promo terms
(esigned PDFs)

Expenses: travel miles/time fees, receipts, approvals

Actions: add extras, add overtime, propose reschedule, cancel, open
dispute, mark complete

F\) Collaboration features (in chat, zero friction)

Action cards render inline (mobilefriendly):

Propose new time → selector → both see Accept/Decline

Add extra (catalog) → confirm & pay in one tap

Overtime (30/60min blocks) → both approve → charge

Upload proofs → gallery preview with Approve/Request changes

Upload finals → mark milestone "Delivered" → triggers release timer

Expense receipt → buyer approve/pay

Share location (general area, not exact address unless both consent)

Safety flag (see §H)

Annotations: On moodboard and shot list items, inline comments +
\@mentions.

Pin message: Keep key messages at top (address, wardrobe, parking code).

Labels & notes: Private labels ("VIP", "High budget", "No reshoots")
visible only to you.

Voice notes (Phase 2) and video call link (Phase 2) if needed---kept
lightweight.

G\) File handling & delivery guardrails

Supported types & limits (MVP):

Images up to 25MB each; Videos up to 500MB; Docs up to 50MB; Zips up to
1GB

Perthread storage cap (e.g., 10GB) with option to add more (paid or Pro
plan)
