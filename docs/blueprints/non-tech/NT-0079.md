§4.1 --- Global Information Architecture & Navigation

Contract: Make it obvious where to go for any task. Keep People vs
Places distinct in IA, keep city & Safe Mode always available, and keep
create actions contextual (buyer vs seller vs host). All navigation
choices must support the Houstonfirst rollout and scale to multicity.

A\) IA Overview (mental model & global state)

Entities:

People (Service Profiles / roles): Model • Photographer • Videographer •
Creator • FanSubscription Creator

Places (Space listings): Studio (can be Verified Studio via Studio Lite)

Global state persistent in nav: City (chip), Safe Mode (toggle), Auth
(account), Search entry.

Discover → Plan → Book → Work → Close → Support flows are reachable from
any context within ≤2 clicks/taps.

B\) Desktop Navigation (top bar + browse)

Top bar (left → right)

Logo → Home (cityaware modules)

Browse (megamenu; hover/click)

People: Models • Photographers • Videographers • Creators •
FanSubscription Creators (18+)

Places: Verified Studios (Lite) (subhub; not a toplevel pillar)

By City: quick list (current city first)

Case Studies

Search (prominent) → opens Role/Listing selector + filters drawer

Context area (autoswitches by user type and current route)

Buyer context: Boards

Seller context: Packages

Host context: Studios (quick link to studio dashboard)

If multirole: a small dropdown to switch the context link (Boards \|
Packages \| Studios)

Messages (badge)

Notifications (bell; grouped)

Create (primary CTA dropdown; contextaware)

Buyer: New Brief (Quick) • New Brief (Full)

Seller: New Package • New Post (case study/availability)

Host: New Studio Listing

Safe Mode toggle (default ON; age confirm required to OFF)

City selector (Houston default; typeahead)

Account menu

Profile switcher (tabs: Model, Photographer, Videographer, Creator,
FanSub Creator, Studio host)

Buyer Dashboard • Seller Dashboard • Studio Dashboard

Verification Center • Payments & Tax • Settings • Help • Sign out

Header behavior

Sticky on scroll; Search and Create remain visible.

When viewing saved searches/boards, a context bar under the header shows
chips for active filters and provides Save / Edit controls.

C\) Mobile Navigation (tab bar + overflow)

Bottom tab bar (5 items)

Home (cityaware feed + modules)

Search (role selector + filters drawer)

Create (FAB; buyer/seller/host contextual)

Messages

Profile (owner view w/ role switcher tabs at the top)

Header controls (mobile)

City chip left; Safe Mode shield icon right (tap → age confirm +
toggle).

Overflow (⋯) menu: Boards, Notifications, Verification, Payments,
Settings, Help.

Sticky action bars

On Profile/Studio detail: sticky Book/Request bar.

On Search: sticky Filters / Save bar.

On Checkout: sticky Pay & Confirm bar.

D\) Search Entry Pattern (everywhere consistent)

Clicking/tapping Search opens a compact selector with six tabs:\
Model \| Photographer \| Videographer \| Creator \| FanSub Creator \|
Studio

Tabs = People (first five) and Places (Studio) --- visually separated
with a divider label.

Filters drawer (right on desktop; fullscreen on mobile) includes:

Common: city/radius, availability window, budget/price, distance,
minimum Rep, Verified/Trusted toggles.

Rolespecific (People): look & comfort (models/creators), specialties &
deliverables (photographers/videographers), promo posting windows
(FanSub Creators), perplatform social metrics (IG/YT/TT/X).

Studio (Places): price/hr, min hours, capacity, creative amenities
(natural light, cyc wall, blackout, ceiling height, loadin/parking,
makeup/dressing room, grip/lighting, etc.), Instant Book, Verified
Studio toggles.

Saved Search: clearly visible Save button with cadence control (daily
batch); confirm sheet explains alerts & Safe Mode behavior.
