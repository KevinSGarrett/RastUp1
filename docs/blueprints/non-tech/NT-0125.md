E\) Calculators (nocode, policyaccurate)

Refund Calculator\
Inputs: role, cancel timing or lateness, price lines
(packages/addons/license/travel), policy bands.\
Output: buyer refund \$; provider clawback \$; platform fee adjustments;
who pays processor fees per policy.\
Oneclick Apply → posts ledger events + issues amended receipt (§4.12).

Delivery/Acceptance Calculator\
Inputs: due date, posted date, items delivered vs promised, acceptance
window, revision rounds left.\
Output: "Eligible for partial refund X%" or "Request revision" path;
sets autoreminder in the thread.

Travel Fee Calculator\
Inputs: geo distance (map link), city override bands from §4.14.2.\
Output: allowable travel fee or adjustment recommendation.

Deposit Claim Calculator (Studios)\
Inputs: evidence, claimed damages types, cap %, expiry.\
Output: capture up to \$Y; requires twoperson approval when \>
threshold.

F\) Standard Workflows (Playbooks)

1\) Reschedule / Change Order

Propose new time; pricing delta autocomputed; both parties accept → SOW
updates → amended receipt.

2\) Buyer Noshow

Confirm arrival logs; provider/studio present?

Policy outcome: buyer forfeits %; provider paid per bands; strike
optional if abuse.

3\) Provider Noshow / Late

Confirm location logs; attempt contact in thread.

Outcome: refund buyer; provider forfeits; strike (T&S); remove Boost
eligibility window.

4\) Late / Partial Delivery

Run Delivery calculator; offer revision first if within rounds;
otherwise partial refund per matrix.

5\) Quality Dispute

Require 3 examples & rationale; compare to SOW.

If subjective with adequate delivery, educate + deny; if objective miss,
partial refund.

6\) Studio Access Failure

Validate instructions sent and code logs; if host fault → full/partial
refund; deposit void.

Offer Relocation (city curated list) + goodwill credit (config caps).

7\) Deposit Claims (Studio)

Require timestamped photos, house rules match; capture up to cap;
disputed? move to Neutral Review.

8\) Duplicate/Overcharge

Verify processor logs; refund excess; issue amended receipt.

9\) Chargebacks

Freeze related payouts; assemble evidence pack (receipts, chat,
acceptance logs, deliverables hashes, location fuzz logs); submit; track
outcome; if lost, apply platform loss policy.

10\) LBG: Talent ok / Studio failed (or vice versa)

Prorate group: refund failed leg; preserve valid leg; issue Group
amended receipt.

G\) Neutral Review Panel (Ambiguous / Highimpact)

Composition: 2 Support seniors + 1 crossfunction (T&S/Finance) →
majority decision.

Inputs: timeline, evidence, calculators' outputs, policy excerpt,
customer narratives.

Outcomes: uphold/modify/overturn; optional goodwill credit; document
rationale; update Reputation and strikes if applicable.

H\) Credits & Goodwill (Guardrails)

Goodwill credits are marketingfunded, scoped to city/role, with percase
and peragent monthly caps (from §4.14.2).

Autoexclusion: users with active strikes or abuse patterns.

On apply: credit line appears in receipt; expiry date; cannot be
withdrawn as cash.

I\) Automation & Triggers

Autoopen case: missed checkin/out, no delivery by T+X hours,

Autosuggest macros: populate opening message using case type & policy;
agent reviews/edits.

Autoreminders: revision requested → followup in 48h; acceptance window
about to expire.
