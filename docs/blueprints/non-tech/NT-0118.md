E\) Rollouts, Scheduling & Guardrails

Staged rollout: 1% → 10% → 50% → 100% (by city/role/cohort); automonitor
KPIs (dropdown presets).

Scheduling: publish windows (start/end), local timezone awareness.

Guardrails: block publish if validations fail (e.g., provider fee \<
floor) or if simulated KPI deltas exceed thresholds (e.g., est. takerate
−\> −3%+).

Rollback: oneclick revert; auto cachebust; notify affected modules
(webhooks).

Approvals: twoperson approval required for fees, promo pricing, tax
category map, nexus list, NSFW visibility.

Dryrun: compute outcomes (no apply) for 24h; log deltas to admin
dashboard.

F\) Admin UI (Editor)

Form + JSON dual view; live validation; inline hints; example payloads.

Diff viewer (before/after) with reason code & ticket link.

Impact preview: sample searches/rankings, bookingfee math, promo
inventory fill.

Scope picker: breadcrumb of inheritances; "show effective value" at any
scope.

History & rollback: timeline of publishes; pin "golden" configs.

G\) Service Consumption & Caching

Read path: services request effective_config(scope, key); library
resolves precedence + local cache.

Caching: inmemory TTL + eventdriven invalidation (pub/sub); 99p read
latency \< 5ms.

Bootsafe: services start with last known snapshot; failopen defaults for
noncritical keys; failclosed for safety/money keys.

Schema shared package to avoid drift; CI blocks breaking changes.

H\) Observability & Alerts

Config read metrics: hit/miss, stale fetch, latency; perservice
dashboards.

Drift detection: compare service snapshot vs registry head; alert on
mismatch.

Anomaly rules:

"Fee change \> 1% absolute in 24h across \>2 cities"

"NSFW visibility toggled in Safe Mode city"

"CPC floors moved \> \$0.20 in 1h"

Slack/email alerts on publish/rollback for sensitive domains.
