E\) Context Switching & Role/Host Awareness

Profile switcher (Account menu on desktop; top segmented control on
mobile Profile) lists all enabled roles and Studio host if the account
has at least one studio listing.

Context link in the header (Boards \| Packages \| Studios) automatically
reflects the most relevant workspace for the current route; quick
dropdown to change it.

Create menu always matches the current persona: posting a brief (buyer),
creating a package/post (seller), or adding a studio (host).

F\) Where key actions live (global "where do I...?" map)

View & edit my offerings (People): Account ▸ Seller Dashboard ▸
Packages/Portfolio/Calendar.

Create a studio listing (Places): Create ▸ New Studio Listing → wizard;
manage via Studio Dashboard.

Post a brief (buyers): Create ▸ New Brief (Quick/Full); invite or
broadcast from the brief.

Attach a studio to a shoot: From Brief or Talent Checkout, tap Attach a
Studio (drawer) → returns a Linked Booking Group summary.

Find inspiration: Browse ▸ Case Studies; deep link to People & Places.

Get help: Thread ▸ Get help (contextual flows) or Help Center.

G\) Safe Mode & Gating (global)

Default ON for guests and new users.

People: FanSubscription Creator and profiles with nudity tiers beyond
"Lingerie/Swim" are hidden when Safe Mode is ON.

Places (Studios): never gated by Safe Mode (studio media must remain
SFW).

Emails/push: never include 18+ previews; 18+ content visible inapp only
with Safe Mode OFF.

H\) CrossLinks & Shortcuts (reduce path length)

Profile → Invite opens Smart Invite prefilled with role, city, date
window.

Talent profile with linked studio shows "Owns/Has Studio" chip → deep
link to the Studio detail.

Studio detail shows Host card → deep link to host's People profile(s).

Boards cards expose Compare, Invite, Attach Studio, Book shortcuts.

Case Study tiles link to both the talent and any tagged studio.

I\) Header Context Bar (chips below header)

Appears on Search, Saved Search, Boards, Brief, Compare.

Shows active filter chips, city/date/time chips, and Saved state; inline
Edit and Clear actions.

On Linked Booking Group checkout, shows a twochip summary: Talent \|
Studio (tap to expand each).

J\) Error States & Auth Walls (consistent pattern)

Authrequired actions (book, message, create) route through /auth with a
return URL; copy explains benefits and Safe Mode.

404 gracefully links to Search by role with city preselected.

403 uses policy copy (e.g., 18+ gating, suspended features); provides
Help link.

K\) Accessibility, Localization & Performance (global baselines)

AA: keyboard navigable menus & drawers; ARIA on tabs/filters; obvious
focus states; colorcontrast linting in CI.

Localization: strings via i18n; USEnglish at launch; localeaware
currency/date.

Performance: routebased code splitting; image lazyload; deferral of
OAuth SDKs until needed; search TTI \< 2.5s on 3G Fast.
