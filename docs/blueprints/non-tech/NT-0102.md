G\) Safety, Privacy & Abuse Controls

Autoredact obvious phone/email/payment handles prebooking; after
confirmation, allow a masked relay number/email (Phase 2) or keep inapp
only at v1.

Offplatform payment detection → inline reminder + link to Protection
benefits; repeat attempts throttle new thread creation.

Harassment & slur filter → soft block + "Report" affordance; P0 safety
escalations surface hotline.

Attachment guards → virus scan, MIME/type check, SafeMode preview
compliance; large files rejected with Delivery guidance.

Block/Mute other party; Report with categories (misrepresentation,
harassment, payment circumvention, explicit content).

Privacy: toggle Read receipts ON/OFF per user (default ON); Last seen
off by default.

H\) Inbox (List) & Search

Unified inbox with folders: All, Action needed, Booked, Archived.

Sort: Recent activity (default) • Unread first • By pipeline stage.

Filters: Role (if multirole), Studio (if host), City, Has booking, Has
docs pending, Has delivery pending.

Search within inbox: counterpart name, project title, booking ID.

Archive & Pin threads; archive autoreopens on new activity.

I\) LBGAware Collaboration

Group bar on talent or studio booking thread shows the partner booking
card with status and a "Broadcast to both" toggle.

Broadcast sends a message to both threads with a banner "Sent to both
bookings"; replies arrive in respective threads (kept separate for
policy/payout reasons).

Reschedules launched from either thread attempt to update both bookings;
system explains outcomes if only one can move.

Docs & delivery remain per booking; crosslinks appear in the group bar.

J\) Timezones, Localization & Calendars

All timestamps show local time for each user; hover reveals UTC and
other party's local time.

Date/time proposals include both parties' local times in the message
bubble.

ICS attachments created on booking and on accepted reschedules.

K\) Reliability & Performance

Optimistic send with retry; duplicate protection (idempotent keys).

Typing and read indicators; fall back gracefully on poor networks.

Attachment uploads show progress; resume on reconnect.

Message retention: durable; no hard expiry.
